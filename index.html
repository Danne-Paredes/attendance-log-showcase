Attendance Tracker:

Welcome to the upgraded version of Knighted's Attendance Log and the first step in transitioning Knighted towards a more modern and data friendly tracking system!

Input Sheet

Employee Info: Contains a dropdown list (cell C4) which is filterable by location (Cell J4), clicking the magnifying icon will initiate a search with BigQuery. Also found are the employees current location and documentation status as well as a coutdown until everything falls off.
Input Infractions/Documentation: Here is where we input the any new infractions/documentations, hit the enter button on the right to submit.
Active Infractions/Documentations: Any current infractions/documentations can be referenced here. Blue rows are excused and not counted towards escalation and Red rows are to notate Documentations.

Attendance History

Employee Info: Contains a dropdown list (cell C4) which is filterable by location (Cell H4), clicking the magnifying icon will initiate a search with BigQuery. Also found are the employees current location and documentation status as well as a coutdown until everything falls off.
Infractions/Documentations: An employees complete infraction/documentation history can be referenced here. Blue rows are excused and not counted towards escalation, Red rows are to notate Documentations and Greyed-Out rows are for expired infractions/documentations.




Resources

Everything used today can be done within the Google suite of apps! Be aware that while most of it is free, depending on data size you may incur charges through bigquery. Luckily google gives you a free $100 credit your first year which goes a long way. For reference, my year is almost up and I still have around $40 left after all my tinkering to make this app work plus a few other projects throughout the year.

Google Sheets

The "UI" is a Google Sheets file (Microsoft Excel with a Google twist) that also houses the formulas that do the heavy lifting in terms of calculating when infractions warrant an escalation based on the given attendance policy.

Google AppsScript
(GAS)

A mix between Visual Basic and JavaScript, Google AppScript allows users to apply custom codes to their Google Sheets file! This project requred two different codes, one for sending data to another Sheets file and the second for sending out a BigQuery DML and outputting the results on the Sheet!

Google BigQuery

Googles answer to SQL based cloud storage, the inclusion of BigQuery allows us to solve the biggest obstacle we faced with the previous version of the "Attendance Log", data size limits. Google Sheets is a web based file so eventually you will run into the ceiling of what Google and your computer can handle. BigQuery allows us to use Goolges network to handle this issue by only pulling relevant data so as to not to overload the system
